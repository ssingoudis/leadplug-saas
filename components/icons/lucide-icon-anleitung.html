Lucide-Style SVG Icon Framework

Anleitung zum Erstellen eigener Icons im exakten Lucide React Stil — für Gemini, ChatGPT oder andere KI-Tools

Zweck dieses Dokuments

Dieses Dokument definiert die technischen und visuellen Regeln für Custom SVG Icons, die nahtlos neben Lucide React-Icons eingesetzt werden können. Lade dieses Dokument zusammen mit einem Referenz-SVG hoch — die KI erstellt daraus ein passendes Icon.

Technische Pflicht-Spezifikation

Diese Werte sind unveränderlich. Jede Abweichung macht das Icon inkompatibel.

EigenschaftWertErklärung
viewBox"0 0 24 24"Standard für alle Lucide Icons
fill (root)"none"Kein Fill auf dem SVG-Element
stroke"currentColor"Farbe kommt immer vom Parent (CSS)
strokeWidth1.5Exakte Strichstärke — nicht 1, nicht 2
strokeLinecap"round"Runde Linienenden
strokeLinejoin"round"Runde Ecken/Verbindungen
Erlaubte Elementepath rect circle line polylineKeine groups, defs, clipPath, mask
fill auf Kind-ElementenNur "currentColor" für PunkteNur für winzige Dots erlaubt

Visuelles Grid & Proportionen

Das Artwork muss innerhalb von x: 2–22 / y: 2–22 bleiben. 1px Rand auf allen Seiten ist Pflicht — Icons die den Rand berühren wirken abgeschnitten.

Visuelle Regeln

✓ Erwünscht

  • Geometrische, klare Formen
  • Max. 6–8 Elemente gesamt
  • Gleiche Linienstärke überall
  • Erkennbar bei 16×16 px
  • "Skizzenhafter Umriss"-Stil
  • Symmetrisch wo möglich
  • Runde Ecken (rx="1" bei rect)

✗ Verboten

  • fill auf Paths/Shapes (außer Dots)
  • Gradients, shadows, filters
  • Mehr als 8 Elemente
  • Zu viele Details / Texturen
  • stroke-width Variationen
  • Artwork außerhalb x/y 2–22
  • <g>, <defs>, <clipPath>

Prompt-Template für Gemini / ChatGPT

Lade dieses Dokument hoch, füge dein Referenz-SVG hinzu und schicke diesen Prompt:

AUFGABE: Erstelle ein SVG-Icon nach dem Lucide-Style-Framework aus der hochgeladenen Anleitung. REFERENZ-SVG: [hier dein SVG-Code einfügen ODER Bilddatei anhängen] ICON-NAME: [z.B. "Satteldach" / "Flachdach" / "Reihenhaus"] BESCHREIBUNG: [z.B. "Ein Haus mit dreieckigem Giebeldach, Schornstein links, Eingangstür mittig"] PFLICHT-REGELN (aus der Anleitung): - viewBox="0 0 24 24" - fill="none" auf dem SVG - stroke="currentColor" strokeWidth="1.5" - strokeLinecap="round" strokeLinejoin="round" - Artwork nur zwischen x/y 2–22 - Max 6–8 Elemente - KEIN fill auf Paths außer bei Dots (fill="currentColor" stroke="none") - KEINE groups, defs, gradients, shadows OUTPUT: Nur den SVG-Code, ohne Erklärung.

Korrektur-Prompts

Icon zu detailliert / zu viele Elemente

Das Icon hat zu viele Details. Vereinfache auf max. 5 Elemente. Das Icon muss bei 20×20px auf einem Bildschirm noch erkennbar sein. Alle technischen Specs bleiben unverändert.

Icon nicht erkennbar / zu abstrakt

Das Icon ist nicht erkennbar. Verwende die markantesten visuellen Merkmale: für ein [Haus] wäre das: Dreieckdach + Rechteck-Körper + Tür. Weniger Elemente, aber die wichtigsten. Alle Specs bleiben gleich.

Stroke-Stärke falsch / Fill vorhanden

Fehler im Output: strokeWidth muss exakt 1.5 sein. Kein fill auf Paths erlaubt (außer bei circle-Dots: fill="currentColor" stroke="none"). Bitte korrigieren, alle anderen Specs bleiben.

Integration in React

Neues Icon in das Projekt einbinden — genau 2 Schritte, eine Stelle:

Schritt 1 — Neue Datei anlegen

Datei: components/icons/custom-[name].tsx

import { Icon, type CustomIconProps } from './_base'

export function MeinNeuesIcon(props: CustomIconProps) {
  return (
    <Icon {...props}>
      {/* SVG-Elemente aus Gemini hier einfügen: */}
      <path d="M..." />
      <rect x="..." ... />
    </Icon>
  )
}

Schritt 2 — In Registry eintragen

Datei: components/icons/index.ts — 3 Zeilen ergänzen:

// oben: import
import { MeinNeuesIcon } from './custom-[name]'

// im export-Block:
  MeinNeuesIcon,

// in CUSTOM_ICON_REGISTRY (Key + Komponente + Label zusammen):
  custom_mein_neues_icon: { component: MeinNeuesIcon, label: 'Lesbarer Name' },
Wichtig: Key muss mit custom_ beginnen — nur Kleinbuchstaben und Unterstriche. Label und Icon-Picker-Anzeigename werden automatisch aus dem Registry-Eintrag abgeleitet. Nichts muss an anderer Stelle eingetragen werden.
