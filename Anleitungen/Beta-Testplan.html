LeadPlug — Beta-Testplan

Software-Test vor dem Launch

Ziel: Ein vollständiger Testtag, an dessen Ende du (a) sicher bist, dass alles funktioniert, und (b) genau weißt, wie jede Funktion arbeitet — bereit fürs Loom-Video + Invites. Von oben nach unten durchgehen, jedes Kästchen ankreuzen. erwartetes Ergebnis. Passt etwas nicht → Kästchen leer lassen, Notiz danebenschreiben.

Datum:  Tester:  Build/Version: 

0.Vorbereitung (einmalig, vor dem Testen)

Zwei Browser-Profile: eines als Agentur (Dashboard), eines als Endkunde (Inkognito) — beide Seiten gleichzeitig testen.
Frische, externe E-Mail-Adresse parat (NICHT dein Stamm-Konto) für den Registrierungs-Test.
Echte Test-Website zum Einbetten (leere HTML-Datei o. Ä.) für das iFrame-Embed.
Geräte: Desktop + Handy (iOS und/oder Android), optional Tablet.
Browser: mindestens Chrome + Firefox + Safari durchspielen.
Webhook-Testziel: webhook.site in einem Tab offen (Test-URL + zeigt eingehende Payloads).

1.Registrierung, Login & Onboarding

app.leadplug.de/signup öffnen Registrierungsformular, sauber in hell + dunkel.
Mit frischer E-Mail registrieren Bestätigungsseite erscheint, Adresse bricht nicht um.
Bestätigungs-Mail kommt an Posteingang (nicht Spam), Link klickbar.
Link klicken → landet direkt im Dashboard NICHT localhost! Beweist Supabase-Site-URL + NEXT_PUBLIC_BASE_URL auf Prod.
Erst-Login: Kontoname-Modal (geblurrt, nicht wegklickbar) Name (min. 2 Zeichen), Enter speichert, Dashboard frei.
Ausloggen → Login. Wieder einloggen mit E-Mail + Passwort Dashboard.
Falsches Passwort klare Fehlermeldung, kein Absturz.
Eingeloggt Root / aufrufen leitet ins Dashboard.
Ausgeloggt /dashboard/leads direkt aufrufen leitet auf Login (kein Zugriff ohne Auth).

2.Konto-Verwaltung /dashboard/account

E-Mail-Feld ist schreibgeschützt (grau).
Kontoname ändern → beim Verlassen (Blur) automatisch gespeichert „Gespeichert"-Status.
Name-Änderung spiegelt in der Sidebar (nach Refresh).
Passwort: < 8 Zeichen → Fehler; zwei verschiedene → Fehler; gültig → „✓ Gespeichert".
Mit neuem Passwort aus-/einloggen klappt.
Gefahrenzone „Konto löschen" anschauen (Button → Modal verlangt exakten Kontonamen).
(Optional, Wegwerf-Konto) Löschen ausführen Konto + Daten weg, Redirect Login, kein erneutes Login.

3.Dashboard-Navigation & Übersicht

Übersicht /dashboard lädt: letzte Leads + Chart, leeres Konto zeigt „Noch keine …".
Sidebar: alle Punkte klickbar (Übersicht, Funnels, Leads, Statistiken, Vorlagen, Konto, Billing) jede Seite lädt fehlerfrei.
Dark-Mode-Schalter sauberer Wechsel, kein weiß-auf-weiß/schwarz-auf-schwarz irgendwo.
Benutzer-Menü: E-Mail sichtbar, Logout funktioniert.
Schmales Fenster / Handy Sidebar klappt sinnvoll, nichts abgeschnitten.

4.Funnel-Verwaltung /dashboard/funnels

„Neuer Funnel" → Auswahl aus Vorlage oder leer.
Aus Vorlage: Vorlage wählen → Namen abfragen → Funnel erstellt + öffnet im Editor.
Leeren Funnel erstellen öffnet mit Grundgerüst.
Duplizieren exakte Kopie, neuer Slug. Webhooks + Tracking-IDs werden NICHT mitkopiert (prüfen!).
Umbenennen (interner Name) gespeichert, in Liste aktualisiert.
Slug ist nach Anlage unveränderlich kein Bearbeiten möglich (schützt Embeds).
Aktiv/Inaktiv schalten inaktiv → öffentliche URL zeigt 404, nicht das Widget.
Funnel-Karte zeigt Lead-Anzahl (nur completed) + Status korrekt.
Löschen → Bestätigungs-Modal gelöscht; Leads bleiben erhalten (Snapshot).
„Live ansehen" ?preview=1 öffnet Funnel, zählt KEINEN Aufruf (Statistik prüfen).

5.Vorlagen-Galerie /dashboard/vorlagen

Galerie lädt, alle Vorlagen als Karten.
Kategorie-Filter filtert korrekt, „Alle" zeigt alles.
Vorschau-Modal: spielbarer Funnel, scrollt bei hohen Schritten (auch kleiner Laptop).
Aus Vorschau erstellen → Funnel im eigenen Konto.
Vorschau-Spieler löst keine echten Mails/Leads aus.

6.Editor — Tab „Bearbeiten" (Inhalt)

Schritt hinzufügen („+") → Auswahl-Modal mit allen Typen.
Schritte per Drag umsortieren Reihenfolge bleibt nach Speichern.
Schritt löschen / duplizieren / ausblenden ausgeblendet erscheint NICHT im Live-Widget.
Schritt zwischen zwei andere einfügen (Pin-Edge) → richtige Stelle.

Jeden Frage-Typ einmal anlegen + im Canvas testen

Einfachauswahl — A/B/C-Chips, springt automatisch weiter.
Mehrfachauswahl — mehrere wählbar, kein Auto-Advance (Weiter-Button).
Kurztext — Eingabefeld + Placeholder.
Langtext — Textarea.
Schieberegler — Min/Max/Schritt/Einheit, Zahl-Readout.
Datum — Kalender, ausgewählter Tag in Funnel-Farbe (nicht Blau!), heute vorausgewählt.
Zahl — Min/Max/Schritt/Einheit.
Dropdown — Auswahlliste.
Checkbox (einzeln) — Label rechts.
Bewertung — Sterne mit Hover-Vorschau.
Skala (NPS) — 0–N Buttons, Labels links/rechts.
Aussage — kein Input, nur OK/Weiter.
Marker-Stil umschalten (A/B/C · 1/2/3 · keiner · Checkbox) → Live korrekt.

Optionen & Pflichtfelder

Option hinzufügen / umbenennen / per Drag sortieren / löschen → spiegelt im Canvas.
Pflichtfeld setzen → leeres Feld blockiert das Weitergehen im Widget.

Spezial-Schritte

Welcome-Screen — eigener Button-Text, kein Zurück-Button, am Anfang.
Kontaktdaten-Karte (Multi-Field) — mehrere Felder auf einer Karte.
Karten-Feld-Typen: Vorname, Nachname, E-Mail, Telefon, PLZ, Radio, Text, Zahl, Datum.
Pflichtfelder auf der Karte → blockieren Submit wenn leer.
Einwilligung als Checkbox mit Markdown-Link [Datenschutz](https://…) → Link klickbar.
Success-/End-Screen — Dank-Text, optional Antworten-Übersicht (Toggle).

7.Editor — Design (Theme)

Markenfarbe ändern Buttons/Chips/Progress/Kalender übernehmen sie (auch im Live-Widget!).
Textfarbe / Card-BG / Seiten-BG ändern „transparent" lässt Parent-Seite durchscheinen.
Alle 10 Fonts durchprobieren (System, Inter, Poppins, Roboto, Montserrat, Open Sans, Lato, Nunito, DM Sans, Merriweather) jeder lädt + sieht richtig aus.
Eckenradius + max. Breite ändern Live übernommen.
Anzeige-Schalter: Fortschrittsbalken, Schritt-Chip, Überschrift links/mittig je im Live-Widget prüfen.

8.Editor — Tab „Logik" (Sprünge)

An einer Frage eine Regel anlegen → Bedingung(en) + Ziel (spätere Seite oder „Ende").
Operatoren: = / ≠, enthält (Freitext), enthält Auswahl (Mehrfach), > ≥ < ≤ (Zahl/Slider/Bewertung/Skala).
„Alle anderen Fälle"-Fallback setzen.
Nur Vorwärts-Sprünge möglich (Editor bietet nur spätere Schritte).
Verzweigungs-Badges erscheinen in der Schritt-Liste.
Logik-Map: Schritte-Kette + „Ende", Sprung-Bögen, Auto-Fit, Zoom (Strg+Mausrad), Drag-Pan. Karten-Klick öffnet Regel-Modal, Stift springt in „Bearbeiten".
Test-Modus im Editor: Sprünge führen wie definiert (übersprungene Pflichtfelder blocken nicht).
Live-Widget mit Logik = gleicher Pfad wie Test-Modus.

9.Editor — Tab „E-Mails" (Drip)

Default-Mails sichtbar (Kunden-Bestätigung + Tenant-Benachrichtigung).
Neue Mail anlegen (3-Pane: Liste · Editor · Live-Vorschau).
Verzögerung 0 (sofort) und >0 (z. B. 5 Min) — beide Modi.
Empfänger-Modus: Lead · feste Adresse(n) · @me (folgt Konto-Adresse) — je testen.
Variablen einfügen → Chip; {{answer.<feld>}}, Lead-Name/E-Mail/Telefon → in Vorschau aufgelöst (Choice→Label).
Magic-Sections: Antworten-Übersicht, Kontakt-Zusammenfassung, Dashboard-Button → in Vorschau gerendert.
CTA-Button (Label + URL) einfügen; Link via Inline-Popover.
Live-Vorschau mit Mock-Daten UND echtem Lead (Picker); Resize-Handle funktioniert.
Auto-Save (~1,5 s); Wechsel warnt bei ungespeicherten Änderungen.
„Test senden" → Mail kommt an, sieht aus wie Vorschau, taucht in Historie auf.

10.Editor — Tab „Webhooks"

Webhook mit webhook.site-URL anlegen.
URL-Validierung: http:// abgelehnt, localhost/private IP abgelehnt, nur https:// öffentlich erlaubt.
Trigger on_submit (Event submission.completed) anlegen.
Trigger after_page → Page wählen → violettes Badge an der Schritt-Pille.
„Test senden" → Payload landet auf webhook.site.
Secret: einmal voll gezeigt (Copy), danach maskiert; Rotieren testbar.
Signatur-Snippets (Node/Python/PHP) vorhanden + kopierbar.
Logs-Drawer zeigt Versuche mit Status/Code/Response.
Webhook deaktivieren → feuert nicht mehr.

11.Editor — Tab „Einbinden"

Embed-Code (iFrame) angezeigt → Copy-Button funktioniert.
Plattform-Anleitungen vorhanden + verständlich.
Meta-Pixel-ID eintragen → Format (nur Ziffern 5–20) → ungültig abgelehnt.
Google-Ads-Conversion → Format AW-XXXX/Label → ungültig abgelehnt.
Embed-Code tatsächlich in die Test-Website einbauen (Abschnitt 14/21).

12.Speichern & Persistenz

Funnel speichern (Top-Save) → Erfolg sichtbar.
Seite neu laden → alles exakt wie gespeichert (nichts verloren).
Undo/Redo funktioniert.
Nach Speichern bleiben Page-IDs stabil → after_page-Webhook bleibt korrekt gebunden.
Mehrere schnelle Änderungen + Speichern → keine Doppelungen/Verluste.

13.Live-Widget — Rendering & Navigation /[slug]

Funnel-URL anonym öffnen → lädt schnell, sauber zentriert.
Welcome-Screen (falls vorhanden) → Start-Button.
Fortschrittsbalken bewegt sich; Schritt-Chip zeigt Nummer (falls aktiviert).
Einfachauswahl → Klick springt automatisch (~250 ms).
Alle anderen Typen → Weiter/OK nötig, kein zu frühes Wegschnipsen.
Zurück-Button → eine Frage zurück, vorherige Antwort noch da (auch über Logik-Sprünge).
Jeden Frage-Typ einmal beantworten Eingabe wird angenommen.
Kontaktdaten-Karte: mehrere Felder, Pflichtfeld leer → Weiter blockiert + Hinweis.
Einwilligung: Datenschutz-Link öffnet (neuer Tab), Checkbox Pflicht falls gesetzt.
Letzter Schritt → OK löst Absenden aus (kein separates Kontaktformular).
Erfolgsseite (Dank-Text) ODER Redirect auf redirectUrl (falls gesetzt) nach ~1,5 s.
Antworten-Übersicht auf der Erfolgsseite (falls aktiviert) korrekt.

14.Live-Widget — Geräte, Themes & iFrame

Handy (Hochformat) → alles bedienbar, Buttons groß genug, kein horizontales Scrollen.
Tablet / schmales Fenster → Layout passt sich an (Container-Queries).
Desktop breit → max. Breite greift, zentriert.
Dunkles Theme → Text lesbar, Kontraste ok.
iFrame-Höhe passt sich an jeden Schritt an (kein Abschneiden, kein inneres Scrollen) — auf echter Test-Website prüfen.
Font lädt im iFrame (kein Flackern auf System-Font).

15.Submit & Lead-Erfassung (End-to-End)

Funnel als Endkunde komplett ausfüllen + absenden.
Lead erscheint im Posteingang /dashboard/leads (neueste oben).
Kontaktdaten korrekt abgeleitet: Name, E-Mail, Telefon, PLZ.
Alle Antworten gespeichert + in Detailansicht mit Fragen-Labels (nicht rohe Keys) + Choice-Labels.
Anonymer Submit (Funnel ohne Kontaktfelder) → erscheint trotzdem als completed Lead.
Leads verschiedener Funnels → richtig zugeordnet (Funnel-Name stimmt).

16.Partial-Submissions & Abbrecher

Funnel anfangen, E-Mail eingeben, abbrechen → nach ~10 Min im Tab „Abgebrochen — mit E-Mail".
Anfangen ohne Kontaktdaten + abbrechen → nicht im Posteingang, aber als Aufruf in Statistik.
Abbrecher-Webhook (falls konfiguriert) feuert nach Cooldown (~10 Min) einmal (nicht doppelt).
Neuer Tab = neue Session; gleicher Tab nach Reload = Session fortgesetzt (Antworten noch da).

17.Sicherheit, Bot-Schutz & Fehler-Routen

Inaktiver Funnel /[slug] → 404/„nicht verfügbar".
Nicht existierender Slug → 404.
Quelltext einer Funnel-Seite prüfen → KEINE internen Felder (notificationEmail, Billing) sichtbar.
Honeypot: Bot-Submit mit gefülltem Honeypot → Server 200, aber kein Lead in DB.
Doppelter Submit (Doppelklick / Reload) → nur ein Lead, Mails/Webhooks einmal (Idempotenz).
Bewusst kaputte URL → freundliche Fehlerseite, kein Stacktrace.
Als Konto A keine Daten von Konto B sichtbar (mit zweitem Wegwerf-Konto checken).

18.Lead-Posteingang /dashboard/leads

Leere Ansicht (neues Konto) → „Noch keine Leads eingegangen".
Listen- und Board-Ansicht umschalten → beide zeigen die Leads.
Status ändern: offen → kontaktiert → abgeschlossen → bleibt nach Reload.
Interne Notiz schreiben → gespeichert (Autosave), nach Reload da.
Filter: Funnel · Status · Zeitraum · Suche → jeweils korrektes Ergebnis.
Sortierung „Neueste zuerst" stimmt.
CSV-Export (nur bei Einzel-Funnel aktiv): Excel (DE, Semikolon+BOM) + Standard (Komma) → beide öffnen → Spalten + Umlaute korrekt, leere Spalten weg, Choice→Label.
Export-Button deaktiviert bei „Alle Funnels" → Tooltip erklärt warum.
Lead-Detail aufklappen → alle Antworten + Kontaktfelder lesbar.

19.E-Mail-Zustellung (echte Mails)

Kunden-Bestätigung kommt beim Endkunden an (nicht Spam).
Tenant-Benachrichtigung kommt bei dir an (notificationEmail).
Brand-Header in Markenfarbe; Footer-Link führt auf app.leadplug.de (nicht tot).
Variablen aufgelöst, keine rohen {{…}}.
Magic-Sections korrekt gerendert.
Absender-Adresse stimmt (verifizierte Domain), Antworten-an sinnvoll.
Verzögerte Mail (delay > 0) kommt zur richtigen Zeit (Cron alle 5 Min).
Mail auf Handy → responsive, lesbar.

20.Webhook-Zustellung (echte Auslieferung)

Nach echtem Submit landet submission.completed auf webhook.site.
Payload enthält: event, contact, answers[] (mit Labels) + answers_flat, available_channels.
Signatur-Header X-LeadPlug-Signature vorhanden.
after_page-Webhook feuert beim Überschreiten der gebundenen Page.
Retry: Ziel liefert 500 → „retrying", Cron wiederholt mit Backoff; 4xx → sofort „failed" (kein Spam).
Logs im Editor spiegeln alle Versuche.

21.Conversion-Tracking (auf der Test-Website)

Embed-Code mit embed.js auf Test-Seite → Widget lädt.
Nach Submit: postMessage 'funnel-submit' feuert (DevTools) → keine PII (nur slug + Pixel-IDs).
Falls GTM: dataLayer-Push leadplug_lead sichtbar.
Falls Meta-Pixel-ID: fbq('track','Lead') feuert (Pixel Helper).
Falls Google-Ads: Conversion feuert.
window.LeadPlug.onLead-Callback (falls genutzt) wird aufgerufen.

22.Statistiken /dashboard/statistiken

Seite lädt mit 12-Monats-Daten.
Aufrufe zählen bei echten Besuchen hoch (nicht bei ?preview=1).
Leads zählen nur abgeschlossene Submissions.
Funnel-Filter → Charts aktualisieren sich.
Donut / Monats-Chart / Aufrufe-vs-Leads-Trend → plausibel, kein NaN/leere Achsen.

23.Billing-Seite /dashboard/billing

Grüne „Offene Beta: kostenlos"-Box sichtbar.
Status-Badge „Kostenlos", Feature-Liste, „nichts weiter zu tun".
Kein Stripe-Zwang, keine Kreditkarten-Abfrage.
(Optional, Test-Modus) Test-Kachel „1 €/Monat" → Checkout öffnet Stripe → Zurück-Banner. Nicht nötig für Beta.

24.Feedback-Widget (im Dashboard)

Floating-Button unten rechts → Modal „Nachricht senden".
Kategorie + Nachricht → Senden → Erfolgs-Bestätigung.
Datei-Anhang: ≤ 3 Dateien, zusammen ≤ 4 MB, nur PNG/JPG/PDF → Über-Limit/falscher Typ abgelehnt.
Feedback-Mail kommt bei dir an (SUPPORT_EMAIL), Anhänge dabei, Reply-To = Absender.
WhatsApp-Link (falls gesetzt) öffnet mit vorausgefülltem Text.
Widget blendet sich im Vollbild-Editor aus.

25.Fehler-, Grenz- & Stress-Fälle

Sehr lange Texte in Eingabefeldern → kein Layout-Bruch.
Sonderzeichen / Umlaute / Emoji in Antworten + Namen → korrekt in DB/Mail/Webhook/CSV.
Leere Pflichtfelder überall → saubere Blockade mit Hinweis.
Netzwerk während Submit kappen (DevTools offline) → „Danke" sichtbar, Retry rettet (Partial-Submission als Netz).
Browser-Zurück im Widget → bleibt im Funnel.
Mehrere Tabs desselben Funnels → unabhängige Sessions, kein Daten-Mix.
Schneller Klick-Spam auf Weiter/OK → kein Doppel-Advance/Doppel-Submit.

26.Cross-Browser & Geräte (Quer-Check)

Chrome (Desktop) — Dashboard + Editor + Widget.
Firefox — dasselbe.
Safari (Mac) — dasselbe, besonders Datum/Animationen.
iOS Safari (Handy) — Widget + Submit.
Android Chrome (Handy) — Widget + Submit.
Dark Mode auf jedem davon stichprobenartig.
