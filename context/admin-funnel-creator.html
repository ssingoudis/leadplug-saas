Übersicht

Was ist der Funnel-Creator?

Der Admin Funnel-Creator ermöglicht es, einen neuen Kunden (Tenant) inklusive Funnel und allen Fragen vollständig über ein Formular anzulegen — ohne direkten SQL-Zugriff auf Supabase. Er ersetzt das manuelle Eintragen per SQL und dient gleichzeitig als Blaupause für den späteren Self-Service-Editor.

Erreichbar über

  • URL: /admin/new
  • Button "Neuer Kunde" im Admin-Header (/admin)

Zwei Aktionen

  • Vorschau — rendert Funnel ohne DB-Schreibzugriff
  • Speichern — schreibt in DB, öffnet /admin/[slug]

Reihenfolge der INSERTs: tenants → funnels → funnel_questions. Bei Fehler in einem Schritt stoppt die API und gibt die Fehlermeldung zurück — die vorherigen Einträge bleiben in der DB. Deshalb: Slug-Eindeutigkeit vorher prüfen, da kein automatischer Rollback.

Vorschau-System

Vorschau ohne DB-Schreibzugriff

Wie es funktioniert

  1. Klick auf "Vorschau"
  2. Kompletter Formular-State wird in localStorage unter Key funnel_preview gespeichert
  3. /admin/preview öffnet sich in neuem Tab
  4. Preview-Seite liest aus localStorage und rendert <Funnel>-Komponente direkt
  5. Gelbes Banner: "Vorschau — noch nicht gespeichert"

Was die Vorschau zeigt

  • Alle Fragen mit Optionen/Slider/Freitext
  • Primärfarbe + Font
  • Alle konfigurierten Texte
  • Standard-Kontaktfelder (Anrede, Name, Telefon, E-Mail)
  • Submit-Button → no-op (kein echter API-Call)
Wichtig: Die Vorschau spiegelt immer den Stand zum Zeitpunkt des Klicks. Änderungen nach dem Klick erfordern einen erneuten Klick auf "Vorschau" — der Tab aktualisiert sich nicht automatisch.
Auto-Generierung

Slug- und Key-Generierung

FeldGeneriert ausLogikManuell überschreibbar
tenant_slug Firmenname Kleinbuchstaben · ä→ae, ö→oe · Sonderzeichen → - Ja
funnel_slug tenant_slug Identisch zum Tenant-Slug (Standard: ein Funnel pro Tenant) Ja
question_key Fragetitel Kleinbuchstaben · ä→ae · Sonderzeichen → _ Nein (unsichtbar)
Option value Option-Label Gleiche Logik wie question_key Nein (unsichtbar)
Slugs werden nur beim Tippen auto-generiert solange sie noch nicht manuell verändert wurden. Nach manueller Änderung wird die Auto-Generierung für das jeweilige Feld deaktiviert.
