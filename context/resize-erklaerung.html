iFrame Höhenanpassung

Wie das Widget dem Parent-Frame seine Höhe mitteilt — kein Scrollbalken, nahtlose Integration

Problem & Lösung

Das Problem

Ein iFrame hat eine fixe Höhe (z. B. 500 px). Wenn der Funnel mehr Inhalt hat — z. B. beim mehrzeiligen Kontaktformular oder dem Danke-Screen — erscheint ein Scrollbalken innerhalb des iFrames. Das sieht unprofessionell aus und stört die UX.

Die Lösung

Das Widget misst nach jedem Rendering seine eigene Höhe und sendet sie per postMessage-API an die Elternseite. Ein Script auf der Kunden-Website empfängt diese Nachricht und passt die iFrame-Höhe dynamisch an — kein Scrollbalken, kein manuelles Eingreifen.

Ablauf in 4 Schritten
1
📐
ResizeObserver
Widget beobachtet den Container. Löst bei jeder Größenänderung aus — Schrittwechsel, Formular, Erfolgsscreen.
new ResizeObserver(
() => sendHeight()
).observe(ref)
2
📤
postMessage senden
Widget sendet die aktuelle scrollHeight an den Parent-Frame. Funktioniert auch domain-übergreifend.
window.parent
.postMessage({
type: 'funnel-resize',
height: ref.scrollHeight
}, '*')
3
👂
Parent hört zu
Ein <script>-Tag auf der Kunden-Website lauscht auf message-Events und filtert nach dem Typ.
window.addEventListener(
'message', (e) => {
if (e.data.type !==
'funnel-resize') return;
// Höhe anpassen …
})
4
↕
iFrame angepasst
Das iFrame-Element erhält die neue Höhe. Nahtloser Übergang bei jedem Schritt im Funnel.
iframe.style.height =
e.data.height + 'px'

Vollständiger Einbettungs-Code für Kunden
<!-- 1. Das iFrame einbetten --> <iframe src="https://domain.de/musterbetrieb-solar" id="funnel-widget" style="width:100%; border:none; display:block; height:500px;" scrolling="no" loading="lazy"> </iframe> <!-- 2. Listener für automatische Höhenanpassung --> <script> window.addEventListener('message', function(e) { if (!e.data || e.data.type !== 'funnel-resize') return; var h = parseInt(e.data.height, 10); if (h > 0) document.getElementById('funnel-widget').style.height = h + 'px'; }); </script>
Ohne Listener: Das Widget funktioniert weiterhin — es behält die fixe Startgröße (500 px) und scrollt bei Bedarf intern. Kein JavaScript-Fehler, keine Unterbrechung des Funnels. Der Listener ist optional, aber empfohlen.
