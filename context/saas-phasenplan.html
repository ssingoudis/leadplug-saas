5
Agentur- & Premium-Features
○ Geplant — Zielgruppe Agenturen
○ Geplant
📡
Conversion-Tracking via postMessage Funnel sendet Events (step_change, answer, submit_success) per postMessage nach außen. Tenant aktiviert optional Bridge-Snippet, das Events an Google Ads / Facebook Pixel weitergibt. 80% der Tracking-Vorteile eines Skript-Embeds bei 10% der Komplexität. Kritisch für Tenants mit bezahlter Werbung
📘
Plattform-Anleitungen mit Screenshots Klick-für-Klick-Anleitungen für WordPress, Wix, Squarespace, Webflow, Jimdo. Senkt Support-Last und hilft nicht-technischen Endkunden über die "Wie binde ich das ein?"-Hürde
🔗
Webhook-Output / Zapier-Integration Leads automatisch in CRM (HubSpot, Pipedrive), Slack, E-Mail-Marketing oder beliebige Zapier-Endpoints weiterreichen. Standard-Erwartung von Agentur-Kunden
👥
Tenant-Subaccounts für Agenturen Eine Agentur betreut mehrere Endkunden → ein Login, mehrere Tenants. Switcher im Header, separate Billing-Verwaltung. Macht LeadPlug für die Zielgruppe attraktiv
🏢
Tenant als Workspace mit mehreren Usern DB-Refactoring: Junction-Table tenant_members(tenant_id, user_id, role) mit Rollen owner/editor/viewer statt 1:1 tenant.auth_user_id. RLS-Policies anpassen. UI: Mitglieder einladen, Rollen verwalten. Geschäftsführer + Bürokraft können denselben Tenant verwalten
🎨
Whitelabel-Logo für Premium-Pläne DB-Felder tenants.brand_logo_url + brand_name. Dashboard zeigt das Tenant-Logo statt "LeadPlug" — feature-gated über billing_model. Agenturen können ihren Endkunden ihre eigene Marke präsentieren
📋
Funnel-Vorlagen pro Branche Quick-Start-Templates für Solar, Wärmepumpe, Bad, Dach, Fenster usw. Tenants müssen nicht bei Null anfangen — reduziert Onboarding-Friction massiv
🎯
Server-Side Conversion API Facebook CAPI + Google Enhanced Conversions: Tracking-Events server-seitig senden, damit sie nicht von iOS-Tracking-Schutz oder Adblockern blockiert werden. Premium-Tier-Feature
wenn Agentur-Fokus etabliert
6
Vision
💭 Ideenpool für später
💭 Vision
🧪
A/B-Testing pro Funnel Zwei Funnel-Varianten parallel ausspielen, Conversion-Rate vergleichen, Gewinner-Variante automatisch übernehmen
🌳
Verzweigungs-Logik Bedingte Folge-Fragen je nach Antwort. "Wenn Heizung = Öl → frage nach Tank-Größe; wenn Gas → andere Frage"
🌐
White-Label-Domain pro Tenant funnel.kundendomain.de statt app.leadplug.de/slug — eigene DNS-Konfiguration, Wildcard-Subdomains, automatisches SSL
🔌
Öffentliche API REST-Endpoints für Funnel-Daten, Submissions, Statistiken — damit Tenants/Agenturen eigene Tools andocken können
📊
Lead-Pipeline (CRM-Light) Kanban-View statt Liste: Spalten "Neu / Kontaktiert / Termin / Abgeschlossen / Verloren". Leads per Drag-and-Drop zwischen Status verschieben. Notizen + Aktivitäts-Log pro Lead. Macht aus dem reinen Lead-Empfänger ein echtes Verkaufstool
📅
Termin-Booking direkt im Funnel Nach Funnel-Submit zeigt sich ein Kalender (Calendly-Style) für direkte Terminbuchung. Massive Conversion-Steigerung — Lead wird sofort zu Termin. Integration mit Google Calendar / Outlook für Verfügbarkeitsabgleich
🤖
KI-Features Lead-Scoring (welche Anfragen sind heiß?), Auto-generierte Funnels aus Branchen-Beschreibung ("Erstelle einen Funnel für Heizungsbau"), Smart-Follow-up-Texte. Hebt das Produkt vom Wettbewerb ab
🧭
Onboarding-Wizard für neue Tenants Schritt-für-Schritt-Tour beim ersten Login: Funnel aus Vorlage wählen → Branding anpassen → Code kopieren. Senkt Time-to-First-Lead drastisch und reduziert Support-Anfragen
📜
Audit-Log / Änderungshistorie Wer hat wann was am Funnel geändert? Besonders kritisch wenn mehrere Team-Mitglieder denselben Tenant verwalten. Lead-Status-Changes ebenfalls protokollieren
📧
Visueller E-Mail-Template-Editor Tenants können Lead-Benachrichtigung und Kunden-Danke-Mail vollständig anpassen — Drag-and-Drop-Blöcke, eigenes Logo, Brand-Farben. Aktuell sind die Templates hardgecodiert
🌍
Mehrsprachigkeit Dashboard + Funnel-Widget in DE/EN/weitere Sprachen. Tenants können pro Funnel die Sprache wählen — Voraussetzung für internationale Agenturen als Kunden
