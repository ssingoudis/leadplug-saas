Submission Flow

Was passiert serverseitig, wenn ein Lead das Formular abschickt?

1
POST /api/submit
Browser sendet Formulardaten an die Next.js API-Route.
{ slug, answers, contact, honeypot }
2
Honeypot-Check
Ist das versteckte Feld ausgefüllt? Menschen sehen es nicht. Bots füllen es automatisch aus.
3
Rate Limit-Check
Max. 3 Submissions pro IP in 10 Minuten. Bei Überschreitung: stiller Reject — identisch wie Honeypot.
4
Payload-Validierung
Sind alle Pflichtfelder vorhanden? Stimmt die Datenstruktur? Fehler → HTTP 400.
5
getTenantConfig(slug)
Supabase: Tenant + Funnel + Fragen laden. Funnel oder Tenant inaktiv → HTTP 404.
lead_price ← tenants.lead_price  // immer server-side!
6
logSubmission() — ZUERST
Submission in Supabase speichern, bevor E-Mails versendet werden. Billing darf nie durch einen E-Mail-Fehler verloren gehen.
submissions: contact, answers, lead_price, ip_address
✉ Kunden-Mail
An: contact.email
Danke-Nachricht + Antwortzeit
Kein Preis, kein PDF
📬 Tenant-Mail
An: tenants.contact_email
Kontaktdaten + Antworten
replyTo = contact.email
Promise.all() — beide parallel, in try/catch — Fehler werden geloggt, nicht geworfen
8
{ success: true }
Widget zeigt den Danke-Screen. Der Endkunde erhält immer eine Erfolgsantwort — auch wenn eine E-Mail fehlschlägt.
Hinweise zu den Schritten
Schritt 2 — Bot erkannt Gibt {success: true} zurück — ohne DB-Eintrag.
Bots dürfen nicht erkennen, dass sie abgelehnt wurden, sonst passen sie ihr Verhalten an.
Schritt 3 — Rate Limit Fail-Open: Bei einem DB-Fehler beim Prüfen wird der Request durchgelassen. Schutz vor Spam, aber kein Blockieren bei Infrastrukturproblemen.
Schritt 6 — Reihenfolge kritisch Die Submission muss in die DB, bevor E-Mails versendet werden. Fällt Resend aus, ist der Lead trotzdem gespeichert und wird abgerechnet.
Schritt 7 — E-Mail-Fehler Schlägt eine E-Mail fehl, wird dies in customer_email_sent bzw. tenant_email_sent geloggt — der Endkunde bekommt trotzdem success:true.
Farben:
Sicherheits- & Validierungscheck
Supabase-Datenbankoperation
E-Mail-Versand via Resend
Erfolgsantwort an den Client
