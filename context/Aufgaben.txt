Deine nächsten Schritte – Schritt für Schritt
Phase 1: Supabase vorbereiten
Schritt 1 – Supabase-Projekt öffnen

Gehe auf supabase.com, melde dich an und öffne dein bestehendes Projekt (oder erstelle ein neues, falls du noch keins hast).

Schritt 2 – Schema einrichten

Im linken Menü auf "SQL Editor" klicken
Oben rechts auf "New query" klicken
Den kompletten Inhalt von context/supabase-schema.sql reinkopieren
Auf "Run" klicken (grüner Button)
Prüfen: Im linken Menü unter "Table Editor" müssen jetzt 4 Tabellen sichtbar sein: tenants, funnel_questions, funnel_options, submissions
Schritt 3 – Seed-Daten einspielen (Claude macht das zuerst)

Bevor du das hier tust, musst du zuerst Schritt 4 machen (Claude die Seed-Datei erstellen lassen). Dann:

Wieder im SQL Editor → "New query"
Inhalt von context/supabase-seed.sql reinkopieren
"Run" klicken
Prüfen: Im Table Editor → tenants müssen 2 Einträge sichtbar sein (demo und demo-waermepumpe)
Phase 2: Umgebungsvariablen einrichten
Schritt 4 – Supabase-Zugangsdaten holen

Im Supabase-Projekt: linkes Menü → "Project Settings" (Zahnrad-Icon ganz unten)
Dann "API" klicken
Folgende Werte notieren/kopieren:
Project URL → das ist dein SUPABASE_URL
service_role Key (nicht der anon Key!) → das ist dein SUPABASE_SERVICE_KEY
Schritt 5 – .env.local Datei ausfüllen

Öffne die Datei .env.local in deinem Projekt (liegt im Stammverzeichnis). Falls sie noch nicht existiert, kopiere .env.example und benenne sie um. Fülle die Felder aus:


SUPABASE_URL=https://deinprojekt.supabase.co
SUPABASE_SERVICE_KEY=eyJ...langer-key...
RESEND_API_KEY=re_xxxxx          ← aus resend.com
EMAIL_FROM=noreply@deinedomain.de
NEXT_PUBLIC_BASE_URL=http://localhost:3000
Den RESEND_API_KEY findest du auf resend.com unter "API Keys".

Phase 3: Claude die Code-Aufgaben ausführen lassen
Jetzt kommt der einfache Teil – du schreibst Claude einfach der Reihe nach:

Nachricht 1:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 2 aus.

(Claude erstellt die Seed-Datei. Danach spielst du die Seed-Daten in Supabase ein – siehe Schritt 3 oben.)

Nachricht 2:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 3 aus.

(Claude stellt getTenantConfig.ts auf Supabase um.)

Nachricht 3:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 4 aus.

(Claude aktualisiert types/index.ts.)

Nachricht 4:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 5 aus.

(Claude baut funnel.tsx – das ist die größte Aufgabe, dauert am längsten.)

Nachricht 5:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 6 aus.

(Claude aktualisiert die API-Route.)

Nachricht 6:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 7 aus.

(Claude aktualisiert die E-Mail-Templates.)

Nachricht 7:

Hallo Claude, lies CLAUDE.md und führe Aufgabe 8 aus.

(Claude löscht tote Dateien und prüft den Build.)

Wichtig: Immer eine Aufgabe nach der anderen. Nicht mehrere gleichzeitig – jede Aufgabe baut auf der vorherigen auf.

Phase 4: Testen
Schritt 6 – Lokalen Server starten

Im Terminal (in VS Code unten auf "Terminal" klicken):


npm run dev
Schritt 7 – Widget im Browser öffnen

localhost:3000/demo → Solar-Funnel (aus Seed-Daten)
localhost:3000/demo-waermepumpe → Wärmepumpen-Funnel
Schritt 8 – Formular komplett ausfüllen und absenden

Danach prüfen:

Supabase → Table Editor → submissions → muss 1 neuer Eintrag sichtbar sein
Deine E-Mail-Inbox → Danke-Mail sollte angekommen sein
Betreiber-Mail-Inbox (aus contact_email des Tenants) → Lead-Benachrichtigung
Schritt 9 – Billing-View prüfen

Im Supabase SQL Editor:


SELECT * FROM monthly_billing;
→ Muss 1 Zeile mit dem Submission-Count und dem berechneten Betrag zeigen.
