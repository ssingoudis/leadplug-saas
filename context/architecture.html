1 · System-Map — die 3 Welten

eine Agentur, drei Bühnen
① Tenant-Backend

Dashboard

② Funnel-Builder

Editor

3-Pane WYSIWYG-Builder. Drag-Reorder, Live-Preview, Theme-Tab, alle Element-Types. Erzeugt Pages + Fields in der DB.
Route: /dashboard/funnels/[slug]/edit
Modus: Inhalt · Design · Logik (bald) · E-Mails · Einbinden
Komponente: EditorShellV2
→
③ Live-Widget

iFrame-Embed

Was Endkunden sehen. Eingebettet auf den Websites der Agentur-Endkunden. Sammelt Antworten, sendet Leads.
Route: /[slug]
Auth: anonym (Service-Key am Server)
Komponente: TenantFunnelClient → Funnel

3 · Datenfluss — vom Tipper bis zum Lead

was passiert nach jedem Tastendruck
1
User Öffnet app.leadplug.de/[slug]
Widget Generiert session_id (sessionStorage), ruft /api/track-view
DB funnel_view_logs Insert + Counter++
2
User Beantwortet Fragen schrittweise
Widget Alle 600ms debounced onAnswersChange → /api/track-progress
DB submissions UPSERT auf session_id (completed_at = NULL)
3
User Klickt finalen Submit (oder Auto-Finish im Skip-Mode)
Server /api/submit: Validation · Rate-Limit · UPSERT mit completed_at = NOW()
Side-Effects 2 Mails (Customer + Tenant) · (bald) Webhook-Delivery
4
Widget Zeigt Success-Page
Optional Wenn redirect_url gesetzt: nach 1.5s window.location.replace()
Tenant Lead erscheint in /dashboard/leads im „Completed“-Tab

9 · Design-Decisions — was historisch geprägt ist

die Eigenheiten und ihre Begründung
Multi-Tenancy via tenant_members + RLS
Junction-Table macht Multi-User pro Tenant einfach. Postgres-Helper-Funktionen current_tenant_ids() sichern jede Query.
Pages-Delete-Replace beim Save
CASCADE-Delete + Re-Insert ist robuster als Diff-Logic für ordered Lists. Editing-Lücke ~1-2min beim Vercel-Deploy bewusst akzeptiert.
kind-Diskriminator statt 3 Type-Arrays
Welcome / Question / Custom leben in einer Liste mit kind-Flag. Sortierung bleibt 1:1 mit DB-sort_order, kein Sync-Problem.
Widget = 1 Datei, hands-off
Doppelnutzung Live + Builder. Splitting würde subtile Render-Drift verursachen. Edits brauchen explizite Freigabe.
Service-Key nur in 6 dokumentierten Ausnahmen
Defense-in-depth. Alle Tenant-CRUDs via User-Client + RLS. Service-Key nur für anonyme APIs (submit, track-view) + System-Operationen.
Partial-Submissions: UPSERT auf session_id
Pricing-Hebel: Abbrecher mit Email zählen als Lead. UPSERT statt INSERT macht Network-Retries idempotent.
Icons komplett raus (Aufg. 34)
A/B/C/D-Letter-Chips als Default. IconPicker mit Custom-SVGs war Branchen-Relikt der Pre-LeadPlug-Phase. Picture-Choice kommt on-demand.
Auto-Advance nur single_choice
Multi-Choice/Rating/Statement brauchen Reflection-Zeit. Welcome + Custom haben eigene OK-Button-Semantik (Welcome: Button-Label-editable; Custom: Validation muss greifen).
Container-Queries (@container) statt Media-Queries
iFrame-Embed-Breite ≠ Viewport-Breite. @md reagiert auf die Card-Breite, nicht aufs Browser-Fenster — funktioniert in Sidebars + schmalen Embeds.
funnels.slug nach Anlage unveränderlich
Sonst würden iFrame-Embeds auf Tenant-Endkunden-Websites brechen. Builder-UI enforced das.
